HTML Elements

HTML (Hypertext Markup Language) elements are the building blocks of web pages, defining thet structure and content of documents displayed in web browsers. These elemets are represented by tags encolsed in angle brackets (<>), with most elements consisting of an opening tag, content, and a closing tag.

  • Opening Tag: An HTML element typically begins with an opening tag, which specifies the type of element being used. For example, <'p'> represents a paragraph element, while <'h1'> denotes a heading element.
  • Content: The content of an HTML element is the information or media contained within the element's opening and closing tags. This content can include text, images, links, links, lists, and other elements nested within.
  • Closing Tag:In most cases, HTML elements require a closing tag to mark the end of the element. Closing tags mirror opening tags but include a forward shalsh (/) before the elements type. For instance, '

    '     indicates the end of a paragraph element.

HTML elements can be nested within one another to create complex document structures, with each element contributing to the overall layout and functionality of the webpage. Understanding how to use HTML elements effectively is essential for web developers to create well-structured, accessible, and visually appealing web pages.

Introduction to HTML5, CSS and JavaScript
HTML5 (Hypertext Markup Language):

HTML is the standard markup language for building web pages, and HTML5 represents its latest version, bringing forth enhancements and new features. It offers tags and attributes to define diverse elements on webpage, encompassing headings, paragraphs, images, links, forms, and beyond. HTML5 introduces semantically rich elements like <'header'>, <'nav'>, <'article'>, <'section'>, <'footer'>, enhancing the structural clarity of web content.

CSS (Cascading Style Sheets):

CSS is used to style and layout the visual presentation of HTML documents. It allows you to control the colors, fonts, spacing, and positioning of elements on a web page. CSS is applied either internally within an HTML document, externally in a separate stylesheet file, or inline within HTML tags.

JavaScript

JavaScript is a versatile scripting language pivotal for imbuing web pages with dynamic and interactive functionality. It empowers developers to manipulate HTML content and structure, process user input, and facilitate communication with servers via Ajax. Widely employed for crafting diverse features such as sliders, form validation, pop-up messages, and beyond, JavaScript offers immense flexibility. It can be seamlessly integrated directly within HTML documents or linked externally, providing developers with various deployment options tailored to their project needs.

10 Advanced Git Commands
1. Add & Commit

You have added and committed the files this way multiple times, but what if I tell you it can be done in one line with the `-am` flag?

$ git add . $ git commit -m "new project"

Try this instead, and it will add the file changes and create the commit using the message.

$ git commit -am "new project" [master 17d7675] new project 4 files changed, 2 intersections(+), 1 deletion(-) 2. Amend

You can rename your current commit message using the `--amend` flag and write the new message. This will help you with accidental messages.

$ git commit --amend -m "Love" [master 7b7f891] Love Date: Mon Jan 22 17:57:58 2024 +0500 4 files changed, 2 insertions(+), 1 deletion(-)

You can include additional changes to the current commit before pushing it to the remote repository. To do so, you need to add the file changes and then commit using the `--amend` flag. To retain the previous commit message, simply use the `--no-edit` flag.

$ git add . $ git commit --amend --no-edit [master f425059] Love Date: Mon Jan 22 17:57:58 2024 +0500 6 files changed, 2 insertions(+), 34 deletions(-) 3. Revert

To undo a commit in Git, you can use the `revert` command. However, this command does not remove any commits. Instead, it creates a new commit that undoes the changes made by the original commit. We will use the `log` with `--oneline` flag to view commit history in a shorter form.

f425059 (HEAD -> master, origin/master) Love 8f184d5 first commit To revert to a previous commit, we use the `git revert` command followed by the commit ID. This creates a new commit with the changes from the previous commit.
